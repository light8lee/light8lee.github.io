# Personal Homepage Blog Redesign

Date: 2026-07-04

## Goal

Rebuild the old GitHub Pages blog as the current personal homepage at:

`https://light8lee.github.io/`

The site should work as a modern static personal blog that can load Markdown articles, host static pages, preserve selected old posts, and support lightweight animation or interactive demos.

## Deployment Model

The desired public URL is a GitHub Pages user site. GitHub Pages requires the repository that serves this URL to be named:

`light8lee.github.io`

The current repository is `light8lee/taotaotao.github.io`, so the implementation can be prepared locally here, but the remote repository must later be renamed or migrated to `light8lee.github.io`.

Deployment configuration:

- Remove the old `CNAME` that points to `light8lee.pub`.
- Set `url` to `https://light8lee.github.io`.
- Set `baseurl` to an empty string.
- Use root-relative or Jekyll-relative links that work from the user-site root.
- Keep the site compatible with GitHub Pages branch deployment.

## Site Architecture

Use Jekyll because the existing repository is already a Jekyll site and GitHub Pages supports it directly.

Primary structure:

- `index.html`: personal homepage with a concise intro, latest writing, selected projects, and visual work.
- `_posts/`: Markdown blog posts.
- `_layouts/`: reusable page, post, and project layouts.
- `_includes/`: shared header, footer, navigation, post cards, and media components.
- `pages/`: standalone pages such as About, Archive, Projects, and Notes.
- `assets/`: images, papers, JavaScript, CSS, and other static files.
- `assets/posts/scpo/`: imported SCPO image-series assets.
- `assets/js/animations/`: reusable animation modules or per-page demo scripts.
- `demos/`: optional standalone static HTML demos when a page needs a self-contained experience.

Old posts from 2016-2017 should remain available, but the homepage should emphasize current work.

## Content Model

The site should support these content types:

1. Blog posts
   - Markdown source in `_posts/`.
   - Front matter for title, date, tags, summary, cover image, and optional external references.

2. Project or visual essays
   - Markdown pages that can embed image sequences, source notes, and external paper links.
   - Useful for research explainers, Xiaohongshu-style image series, and technical visualizations.

3. Static pages
   - About, archive, projects, and link pages.
   - These should be plain Jekyll pages rather than generated by a custom build system.

4. Interactive demos
   - Static JavaScript only.
   - Prefer Canvas, SVG, CSS animation, or small vanilla JS modules.
   - No server-side runtime should be required.

## Visual Direction

The site should feel like a personal research and engineering notebook, not a marketing landing page.

Design qualities:

- Clean, readable, and restrained.
- Strong typography for Chinese and English mixed text.
- Comfortable long-form reading.
- Stable image layouts for vertical image series.
- Clear navigation between writing, projects, archive, and about pages.
- Light animation used to clarify state or add life, not distract from reading.

Avoid:

- Heavy decorative gradients.
- Large marketing-style hero sections.
- Overly card-heavy layouts.
- Old third-party scripts that no longer work, such as legacy share widgets or obsolete comment embeds.
- Hard-coded links to `light8lee.pub`.

## Migration Requirements

The implementation should repair current link problems:

- Remove or replace all links that rely on the old custom domain.
- Avoid `http://` asset URLs.
- Replace old external CDN dependencies where practical.
- Ensure CSS, images, posts, and archives load correctly from `https://light8lee.github.io/`.
- Keep old post URLs stable where possible so old content remains reachable after migration.

- Search indexing service.
- Comment system.
- Custom domain recovery for `light8lee.pub`.

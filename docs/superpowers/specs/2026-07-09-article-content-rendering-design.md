# 文章内容渲染优化设计

## 目标

统一优化网站所有文章内容页：

1. 修复伪代码中的公式无法渲染问题；
2. 将所有普通代码块显示为独立、可横向滚动的代码框；
3. 根据文章一级至三级标题生成左侧目录，支持快速定位；
4. 在移动端将目录折叠为“本文目录”面板。

## 范围

本次改动覆盖所有使用 `_layouts/post.html` 的文章，不局限于“大模型基础知识”合集。现有文章正文仍以 Markdown 为唯一内容源。

不改变 Mermaid 图表的渲染方式，不引入 GitHub Pages 不支持的自定义 Jekyll 插件，也不改造集合列表页。

## 方案选择

采用构建产物上的前端渐进增强：

- Jekyll 继续输出原有文章 HTML；
- 浏览器端脚本扫描标题并生成目录；
- CSS 负责双栏布局、代码框滚动和响应式折叠；
- MathJax 继续负责正文数学公式；
- 含公式的伪代码在 Markdown 源文件中改为可渲染的算法步骤结构。

该方案集中修改共享布局和样式，能自动覆盖新旧文章，同时避开 GitHub Pages 对自定义插件的限制。

## 页面结构

`_layouts/post.html` 增加文章阅读容器：

- `aside`：目录区域；
- 主内容区：原有文章头部和正文；
- 移动端目录触发器：默认收起的“本文目录”按钮。

桌面端采用固定左侧目录和居中正文。目录使用 `position: sticky` 随阅读位置保持可见。移动端恢复单栏，目录位于正文上方并以折叠面板呈现。

没有可用标题的文章不显示目录，正文布局自动恢复为单栏。

## 目录生成与定位

目录脚本在 DOM 就绪后：

1. 扫描正文中的 `h1`、`h2`、`h3`；
2. 保留已有 `id`，否则根据标题文本生成稳定锚点；
3. 对重复锚点追加序号，保证页面内唯一；
4. 按标题层级生成嵌套目录链接；
5. 使用 `IntersectionObserver` 高亮当前阅读章节；
6. 点击条目时平滑跳转；移动端同时收起目录。

脚本属于渐进增强。JavaScript 不可用时，文章正文仍保持完整可读。

## 代码块

所有普通 `pre` 统一显示为独立代码卡片：

- 宽度不超过正文容器；
- `overflow-x: auto`，长行只在代码框内横向滚动；
- 保留语法高亮；
- 使用清晰的边框、背景、圆角和内边距；
- 内联 `code` 保持现有轻量样式，不套用代码卡片。

Mermaid 源代码块不按普通代码卡片展示，仍由 Mermaid 转换为图形。

## 公式与伪代码

MathJax 默认跳过 `<code>` 节点，因此不能依靠代码围栏中的 `$...$` 或 `$$...$$` 正常渲染。含数学公式的伪代码改写为语义化“算法步骤框”：

- 普通步骤使用等宽文本；
- 数学变量和公式放在代码节点之外，由 MathJax 正常处理；
- 可执行 Python 代码继续使用 fenced code block；
- Mermaid 源码保持不变。

静态检查扫描普通代码围栏，若其中出现数学公式分隔符则失败，防止后续内容再次引入同类问题。

## 响应式行为

- 宽屏：左侧 sticky 目录，右侧正文；
- 窄屏：单栏布局，目录折叠为“本文目录”面板；
- 移动端目录默认关闭；
- 点击目录条目后自动关闭；
- 目录和代码块均不得造成页面级横向溢出。

## 异常与边界处理

- 无标题：隐藏目录；
- 重复标题：生成带序号的唯一锚点；
- 标题含标点、中文或行内代码：使用规范化文本生成锚点，并提供序号回退；
- 浏览器不支持 `IntersectionObserver`：目录链接仍可跳转，只不进行滚动高亮；
- JavaScript 被禁用：正文和代码仍正常显示；
- 超长公式或代码：仅对应容器横向滚动，不撑宽页面。

## 验证

自动验证包括：

- Jekyll 构建成功；
- 标题锚点唯一且目录链接有效；
- 普通代码块具有独立横向滚动容器；
- 代码围栏中不存在待 MathJax 解析的公式；
- 现有 Markdown 数学、代码和表格检查全部通过。

浏览器验证选取同时包含长代码、MathJax 和 Mermaid 的代表文章，在桌面和移动端检查：

- 目录生成、跳转与滚动高亮；
- 移动端目录折叠和点击后收起；
- 代码框内部滚动及页面无横向溢出；
- 公式正常渲染且无原始分隔符残留；
- Mermaid 图表不受普通代码框样式影响。

# SCPO 08B V4：按原对话重做 AS / AE / Final A

目标：删除 `β`，不要引入额外符号。直接引用原对话里的买机票例子和数值链。

## 08B-1：匹配到软信用 r

标题：小 r 不是拍脑袋来的

画面内容：
- 成功参考：打开 -> 搜索 -> 选航班 -> 购买 -> 支付 OK
- 失败轨迹：打开 -> 搜索 -> 选航班 -> 购买 -> 支付 FAIL
- 表格：
  - 打开携程：M=0.95，匹配，r=0.90
  - 搜索航班：M=0.92，匹配，r=0.84
  - 选择航班：M=0.88，匹配，r=0.76
  - 点击购买：M=0.90，匹配，r=0.80
  - 支付失败：M=0.40，低于阈值，r=0
- 公式：
  - `r = max(0, M - λ) / (1 - λ)`
  - `搜索航班：r = (0.92 - 0.5) / (1 - 0.5) = 0.84`

结论：前 4 步获得正信用，支付失败信用为 0。

## 08B-2：r 怎么进入 AS，为什么 AE 不变

标题：塑形回报只影响 AS

画面内容：
- Step 1：塑形步骤回报
  - `R̃ = R + α · r_SCPO`
  - 搜索航班：`R=0, r=0.84, α=0.5`
  - `R̃ = 0 + 0.5 × 0.84 = 0.42`
- Step 2：锚点状态分组 -> 计算 AS
  - 8 条轨迹中 5 条经过“搜索航班”锚点状态
  - 组内归一化
  - `AS: -0.5 -> -0.2`
- Step 3：AE 完全不变
  - `AE` 来自终局 reward
  - 使用原始 `R`，不用 `R̃`
  - `AE = -0.5 -> -0.5`

结论：SCPO 不能把失败变成成功，只能让正确步骤少挨罚。

## 08B-3：AS / AE 合成 Final A

标题：最终只是一点优势平移

画面内容：
- Before SCPO：
  - `A = AE + ω · AS`
  - `= -0.5 + 0.5 × (-0.5)`
  - `= -0.75`
- After SCPO：
  - `Ã = AE + ω · ÃS`
  - `= -0.5 + 0.5 × (-0.2)`
  - `= -0.60`
- 差值：
  - `Ã - A = ω · (ÃS - AS)`
  - `= 0.5 × 0.3 = +0.15`
- 对 Loss：
  - Loss 函数不变
  - 唯一变化：`A -> Ã`
  - 效果：不过度抑制正确动作

结论：搜索航班仍被惩罚，但惩罚减弱；支付失败仍被强惩罚。

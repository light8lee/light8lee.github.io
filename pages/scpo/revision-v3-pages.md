# SCPO 创新四/五 V3 拆页方案

## 07A 创新四：为什么要重排

核心句：信用位置是稀缺资源，时间顺序贪心会让早期模板步骤先占坑。

示例：
- 成功参考：打开网站 -> 搜索航班 -> 选择 08:00 航班 -> 核对价格 -> 支付成功
- 失败轨迹：打开网站 -> 搜索航班 -> 搜索航班 -> 选择 08:00 航班 -> 支付失败

按时间顺序处理时，前面的“搜索航班”重复项会先拿到参考位置，后面更关键的“选择 08:00 航班”可能拿不到足够信用。

画面重点：
- 左侧画时间顺序贪心，把绿色信用币投给前三个早期动作。
- 右侧画关键步骤没拿到信用，标注“名额被早期模板抢走”。

## 07B 创新四：怎么重排

核心句：先算所有候选匹配，再用长度/位置平衡排序，让前中后段都有机会申请信用。

实现步骤：
1. 构建失败步骤 i 与参考步骤 j 的相似度矩阵。
2. 过滤低于阈值 theta 的候选。
3. 对候选进行重排，不按 i 的时间先后直接贪心。
4. 分配时仍执行单调约束：参考位置 j 必须向前推进，每个 j 最多用一次。

画面重点：
- 5x5 相似度矩阵。
- 候选队列从“时间队列”变成“平衡队列”。
- 最终匹配箭头沿参考轨迹向前，不交叉、不回退。

## 08A 创新五：完整买机票例子

核心句：失败轨迹的前四步可以和成功参考对齐，最后一步仍然失败。

成功参考：
打开携程 -> 搜北京-上海 -> 选 08:00 航班 -> 进支付页 -> 支付成功

失败轨迹：
打开携程 -> 搜北京-上海 -> 选 08:00 航班 -> 进支付页 -> 支付失败

画面重点：
- 两条轨迹并排。
- 前四步用绿色连线标注“语义一致”。
- 最后一步红色分叉，标注“终局失败不改”。

## 08B 创新五：数值怎么变

核心句：`r` 不是手填的奖励，而是由语义相似度矩阵经过阈值过滤、单调匹配和 soft credit 计算出来的。

上半部分先解释 `r` 怎么来：

1. 失败步骤：`选08:00航班`
2. 成功参考步骤：`选择08:00航班`
3. cross-encoder 给相似度：`sim = 0.92`
4. 阈值：`θ = 0.6`
5. 通过阈值且位置单调，所以这个匹配有效。
6. soft credit 可以画成：`r = (sim - θ) / (1 - θ)`
7. 代入：`r = (0.92 - 0.60) / 0.40 = 0.80`

为了和原来的示意数值衔接，也可以在图里用更保守的例子：

- `sim = 0.84`
- `θ = 0.60`
- `r = (0.84 - 0.60) / 0.40 = 0.60`

下半部分再解释 `r` 如何影响 advantage。

示例数值：
- 原始失败轨迹：AE = -0.5
- 某个正确步骤原本 AS = -0.5
- cross-encoder 相似度 sim = 0.84，阈值 theta = 0.60
- soft credit r = 0.6，alpha = 0.5
- shaped bonus = alpha * r = 0.3
- AS: -0.5 -> -0.2
- 若 omega = 0.5
- A before = -0.5 + 0.5 * (-0.5) = -0.75
- A after = -0.5 + 0.5 * (-0.2) = -0.60

画面重点：
- 四层计算卡：similarity、soft credit、AS、A。
- 用数轴画 -0.75 到 -0.60 的小幅右移。
- 强调“仍为负，只是没那么负”。

## 08C 创新五：对训练信号的影响

核心句：模型仍知道这条轨迹失败，但不会把前面的正确动作学坏。

对比：
- GRPO/GiGPO：失败轨迹每一步都被强惩罚，搜索/选择航班也被压低。
- SCPO：支付失败被惩罚，搜索/选择航班只轻惩罚。

画面重点：
- 左侧“普通训练”：整条红线向下。
- 右侧“SCPO”：前四步绿色被托起，最后一步红色仍向下。
- 底部一句话：“惩罚失败结果，保护正确过程”。

# Rope

传统绝对位置编码（如正弦函数编码、可学习位置嵌入）直接加到词向量上，无法自然捕捉相对位置关系。RoPE 的目标是：

> 让注意力计算中的内积 `q_i · k_j` 只依赖于**向量本身和位置差 `(i - j)`**，而不依赖绝对位置。

这个性质写作：

$$
\langle f(q,i), f(k,j) \rangle = g(q,k,i-j)
$$


也就是说，编码后的查询和键的内积是位置差的函数。

## 3. 代码实现 —— 分步讲解

我们用 PyTorch 实现标准 RoPE，并展示如何应用到注意力中。

### 3.1 预计算频率表

```python
import torch
import math

class RotaryPositionalEmbedding(torch.nn.Module):
    def __init__(self, dim, max_seq_len=2048, base=10000):
        super().__init__()
        self.dim = dim
        self.max_seq_len = max_seq_len
        self.base = base
        
        # 计算 θ_m
        inv_freq = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
        # 生成位置序列 [0, 1, ..., max_seq_len-1]
        positions = torch.arange(max_seq_len).float()
        # 外积: (max_seq_len, dim/2)
        angles = torch.einsum("i,j->ij", positions, inv_freq)
        # concat 得到 (max_seq_len, dim) 的 cos 和 sin
        self.register_buffer("cos_cached", torch.cos(angles).repeat_interleave(2, dim=1))
        self.register_buffer("sin_cached", torch.sin(angles).repeat_interleave(2, dim=1))

    def forward(self, x, position_ids):
        """
        x: (batch, num_heads, seq_len, head_dim)
        position_ids: (batch, seq_len) or (1, seq_len)
        """
        # 获取对应位置的 cos, sin
        cos = self.cos_cached[position_ids]  # (batch, seq_len, dim)
        sin = self.sin_cached[position_ids]
        # 应用旋转
        x_rot = x * cos + self._rotate_half(x) * sin
        return x_rot

    def _rotate_half(self, x):
        # 将 x 的后一半取负，并交换前后两半的位置
        x1, x2 = x[..., : self.dim // 2], x[..., self.dim // 2 :]
        return torch.cat((-x2, x1), dim=-1)
```

### 3.2 关键步骤解释

1. **频率计算**：通过 `base^(-2m/d)` 生成 `d/2` 个不同的角速度，高频在前，低频在后。
    
2. **位置角度**：`positions` 与 `inv_freq` 外积得到每个位置每个维度对的旋转角度。
    
3. **`repeat_interleave`**：因为每两个维度共享一个频率，所以 `cos` 从 `(max_seq_len, d/2)` 复制为 `(max_seq_len, d)`。
    
4. **旋转应用**：对于输入 `x`，我们将它视为 `d/2` 个旋转对：

## 4. 在注意力中的使用

实际模型通常在计算 Q 和 K 后分别施加 RoPE，V 不施加：

```python
# 假设 q, k: (batch, num_heads, seq_len, head_dim)
q_rot = rope(q, position_ids)
k_rot = rope(k, position_ids)
# 再计算注意力分数
attn_weights = torch.matmul(q_rot, k_rot.transpose(-2, -1)) / math.sqrt(head_dim)
```


## 5. 为什么 RoPE 效果出色？

- 直接编码**相对位置**，利于捕捉长距离依赖。
    
- 每对维度旋转，不会破坏向量的模长，数值稳定。
    
- 可以与线性注意力等结构兼容。
    
- 通过调整 base 或插值即可实现长度外推。

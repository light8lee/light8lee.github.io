# Rope

传统绝对位置编码（如正弦函数编码、可学习位置嵌入）直接加到词向量上，无法自然捕捉相对位置关系。RoPE 的目标是：

> 让注意力计算中的内积 `q_i · k_j` 只依赖于**向量本身和位置差 `(i - j)`**，而不依赖绝对位置。

这个性质写作：

$$
\langle f(q,i), f(k,j) \rangle = g(q,k,i-j)
$$


也就是说，编码后的查询和键的内积是位置差的函数。
##  数学推导 —— 从二维旋转开始

### 2.1 二维情形

对二维向量 $x=(x_1,x_2)^\top$，位置为 $i$，我们可以乘以一个旋转矩阵：

$$
f(x,i)=R_{i\theta}x
=
\begin{pmatrix}
\cos(i\theta) & -\sin(i\theta) \\
\sin(i\theta) & \cos(i\theta)
\end{pmatrix}
\begin{pmatrix}
x_1 \\ x_2
\end{pmatrix}.
$$

其中 $\theta$ 是预设的角度频率。这样：

$$
f(q,i)^\top f(k,j)=q^\top R_{(i-j)\theta}k.
$$

其中 $R_{(i-j)\theta}$ 只与位置差有关，目标达成。

### 2.2 高维推广

对于 $d$ 维向量，RoPE 将维度两两分组（共 $d/2$ 对），每一对二维使用不同频率的旋转矩阵，频率从高频到低频变化：

$$
\theta_m=\mathrm{base}^{-2m/d},
\qquad m=0,1,\ldots,\frac{d}{2}-1.
$$

**基频 $\mathrm{base}$ 通常取 $10000$（LLaMA 使用 $10000$）。**

完整 RoPE 表达式：

$$
\operatorname{RoPE}(x,i)_{2m:2m+2}
=
\begin{pmatrix}
x_{2m}\cos(i\theta_m)-x_{2m+1}\sin(i\theta_m) \\
x_{2m}\sin(i\theta_m)+x_{2m+1}\cos(i\theta_m)
\end{pmatrix}.
$$

概括：第 $m$ 个二维子空间进行角度为 $i\theta_m$ 的旋转。


这个设计并不是凭空来的，它有三个核心来源：**继承原版正弦位置编码的频谱思想**、**在旋转中让不同维度对负责不同距离**、**获得自然的远程衰减注意力模式**。

### 正弦位置编码的频率来源

原始 Transformer 的正弦位置编码使用

$$
\begin{aligned}
\operatorname{PE}(pos,2i)
&=
\sin\left(pos\cdot 10000^{-2i/d_{\mathrm{model}}}\right),\\
\operatorname{PE}(pos,2i+1)
&=
\cos\left(pos\cdot 10000^{-2i/d_{\mathrm{model}}}\right).
\end{aligned}
$$

其中 $10000^{-2i/d_{\mathrm{model}}}$ 是角频率。RoPE 沿用同样的几何频率谱，但把正弦和余弦组合成二维旋转矩阵。

### 多频率与指数衰减

把 $d$ 维向量拆成 $d/2$ 个二维旋转对。$m$ 小时频率高、周期短，更敏感于近距离变化；$m$ 大时频率低、周期长，负责远程关系。

相邻频率之比恒定：

$$
\frac{\theta_{m+1}}{\theta_m}
=
\mathrm{base}^{-2/d}.
$$

这种对数均匀分布能用有限维度覆盖从局部到长程的频率范围。对 $\mathrm{base}=10000$ 且 $m=d/2-1$，最低频率近似为

$$
\theta_{\min}\approx \mathrm{base}^{-1}=10^{-4},
$$

对应周期约为

$$
\frac{2\pi}{\theta_{\min}}
\approx
2\pi\times 10000
\approx 62832.
$$

### 远程衰减性质

将 $q$ 和 $k$ 分成二维对后，RoPE 内积可展开为不同频率余弦项的叠加：

$$
\left\langle
\operatorname{RoPE}(q,i),
\operatorname{RoPE}(k,j)
\right\rangle
=
\sum_m
\left(q_{2m}k_{2m}+q_{2m+1}k_{2m+1}\right)
\cos\left((i-j)\theta_m\right)
+\text{交叉项}.
$$

当相对距离 $\lvert i-j\rvert$ 增大时，高频分量快速振荡并在期望上相互抵消，主要剩下低频分量，因此注意力会呈现自然的远程衰减。

|设计点|原因|
|---|---|
|使用旋转矩阵|使内积只依赖相对位置|
|每对维度使用独立频率|从多个尺度度量相对距离|
|频率指数衰减|以对数均匀方式覆盖有限维度的频率范围|
|$\mathrm{base}=10000$|继承经典设定，保证唯一性并便于外推|





## 3. 代码实现 —— 分步讲解

我们用 PyTorch 实现标准 RoPE，并展示如何应用到注意力中。

### 3.1 预计算频率表

```python
import torch
import math

class RotaryPositionalEmbedding(torch.nn.Module):
    def __init__(self, dim, max_seq_len=2048, base=10000):
        super().__init__()
        self.dim = dim
        self.max_seq_len = max_seq_len
        self.base = base
        
        # 计算 θ_m
        inv_freq = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
        # 生成位置序列 [0, 1, ..., max_seq_len-1]
        positions = torch.arange(max_seq_len).float()
        # 外积: (max_seq_len, dim/2)
        angles = torch.einsum("i,j->ij", positions, inv_freq)
        # concat 得到 (max_seq_len, dim) 的 cos 和 sin
        self.register_buffer("cos_cached", torch.cos(angles).repeat_interleave(2, dim=1))
        self.register_buffer("sin_cached", torch.sin(angles).repeat_interleave(2, dim=1))

    def forward(self, x, position_ids):
        """
        x: (batch, num_heads, seq_len, head_dim)
        position_ids: (batch, seq_len) or (1, seq_len)
        """
        # 获取对应位置的 cos, sin
        cos = self.cos_cached[position_ids]  # (batch, seq_len, dim)
        sin = self.sin_cached[position_ids]
        # 应用旋转
        x_rot = x * cos + self._rotate_half(x) * sin
        return x_rot

    def _rotate_half(self, x):
        # 将 x 的后一半取负，并交换前后两半的位置
        x1, x2 = x[..., : self.dim // 2], x[..., self.dim // 2 :]
        return torch.cat((-x2, x1), dim=-1)
```

### 3.2 关键步骤解释

1. **频率计算**：通过 `base^(-2m/d)` 生成 `d/2` 个不同的角速度，高频在前，低频在后。
    
2. **位置角度**：`positions` 与 `inv_freq` 外积得到每个位置每个维度对的旋转角度。
    
3. **`repeat_interleave`**：因为每两个维度共享一个频率，所以 `cos` 从 `(max_seq_len, d/2)` 复制为 `(max_seq_len, d)`。
    
4. **旋转应用**：对于输入 `x`，我们将它视为 `d/2` 个旋转对：
    
    $$
    x=[x_0,x_1,x_2,x_3,\ldots].
    $$
    
    配对为 $(x_0,x_1),(x_2,x_3),\ldots$
    旋转公式：
    
    $$
    \begin{aligned}
    x'_{2m} &= x_{2m}\cos\theta-x_{2m+1}\sin\theta, \\
    x'_{2m+1} &= x_{2m+1}\cos\theta+x_{2m}\sin\theta.
    \end{aligned}
    $$
    
    在代码中通过 `x * cos + rotate_half(x) * sin` 完成。  
    `rotate_half(x)` 输出为 $[-x_1,x_0,-x_3,x_2,\ldots]$，对应公式中的符号。

## 4. 在注意力中的使用

实际模型通常在计算 Q 和 K 后分别施加 RoPE，V 不施加：

```python
# 假设 q, k: (batch, num_heads, seq_len, head_dim)
q_rot = rope(q, position_ids)
k_rot = rope(k, position_ids)
# 再计算注意力分数
attn_weights = torch.matmul(q_rot, k_rot.transpose(-2, -1)) / math.sqrt(head_dim)
```


## 5. 为什么 RoPE 效果出色？

- 直接编码**相对位置**，利于捕捉长距离依赖。
    
- 每对维度旋转，不会破坏向量的模长，数值稳定。
    
- 可以与线性注意力等结构兼容。
    
- 通过调整 base 或插值即可实现长度外推。

# 6. 对比其他方案

RoPE 通过显式旋转 $Q$、$K$ 向量实现相对位置；T5 Relative Bias 直接学习每个相对距离桶的标量偏置；ALiBi 则在注意力分数上加入固定线性惩罚。

|对比维度|RoPE|T5 Relative Bias|ALiBi|
|---|---|---|---|
|实现原理|旋转 $Q$、$K$ 向量|注意力分数加可学习标量|注意力分数加固定线性惩罚|
|是否可学习|否，频率预设|是，学习 Logit Bias|否，斜率固定|
|额外参数量|无|约 $(2L-1)\times h$|极少，每个头一个斜率|
|外推能力|较好，通常配合插值或频率缩放|较弱，需要专门设计|较好|
|归纳偏置|复杂的非单调衰减|可学习任意模式|严格单调衰减|
|计算效率|需要旋转运算|简单加法，但要查表|仅加法|
|KV Cache|需保存旋转后的 $K$|偏置直接作用于分数|偏置直接作用于分数|

MaxRL（Maximum Likelihood Reinforcement Learning）研究的是一个看似很小、实则改变训练偏好的选择：当一道题 8 次采样只对 2 次时，训练应该把它当作“信号太弱”略过，还是把难得的正确轨迹放大？MaxRL 的回答是后者。

本文从 MaxRL 的目标函数出发，依次说明它如何重加权难题、如何落到 GRPO 的 advantage 计算，以及训练时应注意的条件。论文：[Maximum Likelihood Reinforcement Learning](https://arxiv.org/abs/2602.02710)（Tajwar et al., 2026）；项目页：[Zanette Labs / MaxRL](https://zanette-labs.github.io/MaxRL/)。

<div markdown="1">
<p class="visual-note-index">01 / Why MaxRL</p>
<h2>难题不该因为稀有而被忽略</h2>

对带 verifier 的数学、代码等任务，一条完整回答通常只得到 `0 / 1` 奖励。若同一题采样 8 次、只成功 2 次，说明模型并非完全不会，而是已经找到少量可复制的正确路径。MaxRL 的核心直觉是：成功率低的题，恰恰更需要把这几条成功轨迹学进去。

这里的“难”指当前策略下的低成功概率，并不等同于题目长度或主观难度。
</div>
</section>

<section class="visual-note" markdown="1">
<figure><img src="{{ '/assets/posts/video-notes/2026-07-12-maxrl/images/02.png' | relative_url }}" alt="MaxRL 论文与训练问题概览" loading="lazy"></figure>
<div markdown="1">
<p class="visual-note-index">02 / Setup</p>
<h2>把一次做对的概率写成 <code>pθ(x)</code></h2>

对输入 `x`，令 `p_θ(x)` 是模型单次采样得到 verifier 正确答案的概率。普通 RL 直接提升平均成功率：

$$J_{RL}=\mathbb{E}_x[p_\theta(x)]$$

MaxRL 则从“观测到正确答案”的最大似然出发，提升：

$$J_{ML}=\mathbb{E}_x[\log p_\theta(x)]$$

二者只差一个 `log`，但这个变化决定了哪些题在训练中更有话语权。
</div>
</section>

<section class="visual-note" markdown="1">
<figure><img src="{{ '/assets/posts/video-notes/2026-07-12-maxrl/images/03.png' | relative_url }}" alt="普通 RL 与最大似然对难题权重的差异" loading="lazy"></figure>
<div markdown="1">
<p class="visual-note-index">03 / Objective</p>
<h2><code>1/p</code> 是偏向难题的来源</h2>

两种目标的梯度分别是：

$$\nabla_\theta J_{RL}=\mathbb{E}_x[\nabla_\theta p_\theta(x)]$$
$$\nabla_\theta J_{ML}=\mathbb{E}_x[\frac{1}{p_\theta(x)}\nabla_\theta p_\theta(x)]$$

最大似然多出的 `1 / p_θ(x)` 会在成功率很低时变大。因此，同样一次“做对”，对低 `p` 的题产生的更新更强；对已经很容易的题，更新不会无限占据训练预算。
</div>
</section>

<section class="visual-note" markdown="1">
<figure><img src="{{ '/assets/posts/video-notes/2026-07-12-maxrl/images/04.png' | relative_url }}" alt="最大似然与 pass at k 的展开关系" loading="lazy"></figure>
<div markdown="1">
<p class="visual-note-index">04 / Derivation</p>
<h2>最大似然其实在优化所有 <code>pass@k</code></h2>

利用对数展开：

$$\log p=-\sum_{k=1}^{\infty}\frac{(1-p)^k}{k}$$

其中 `(1-p)^k` 是连续 `k` 次都失败的概率，即 `fail@k`。对参数求导后：

$$\nabla_\theta\log p=\sum_{k=1}^{\infty}\frac{1}{k}\nabla_\theta\operatorname{pass@}k$$

普通 RL 只对应第一项 `pass@1`；MaxRL 以调和权重同时改善所有 `pass@k`。这正适合“单次不稳定，但多次采样有机会成功”的问题。
</div>
</section>

<section class="visual-note" markdown="1">
<figure><img src="{{ '/assets/posts/video-notes/2026-07-12-maxrl/images/05.png' | relative_url }}" alt="有限 rollout 如何逼近最大似然目标" loading="lazy"></figure>
<div markdown="1">
<p class="visual-note-index">05 / Rollouts</p>
<h2>更多 rollout 改变的不只是方差</h2>

实践中使用有限阶截断：

$$J^{(T)}_{MaxRL}=-\sum_{k=1}^{T}\frac{(1-p)^k}{k}$$

当 `T = 1`，它退化为普通 RL；当 `T → ∞`，它恢复最大似然。于是 rollout 数并非单纯“多采一点、估计稳一点”：增加每题的 rollout 上限，也让实际优化目标更接近最大似然，同时增加采样成本。
</div>
</section>

<section class="visual-note" markdown="1">
<figure><img src="{{ '/assets/posts/video-notes/2026-07-12-maxrl/images/06.png' | relative_url }}" alt="GRPO 中按同一 prompt 分组计算奖励" loading="lazy"></figure>
<div markdown="1">
<p class="visual-note-index">06 / GRPO Setup</p>
<h2>实现从同题 rollout 组开始</h2>

对同一个 prompt 采样 `N` 条 rollout，奖励为可靠 verifier 给出的 `r_i ∈ {0, 1}`。记：

$$K=\sum_{i=1}^{N}r_i,\qquad \bar r=K/N$$

`K` 是本组正确条数，`r̄` 是本题当前成功率。MaxRL 的信息只在这种同题分组内成立：它要根据“这道题已经有多少条成功回答”来决定每条轨迹的权重。
</div>
</section>

<section class="visual-note" markdown="1">
<figure><img src="{{ '/assets/posts/video-notes/2026-07-12-maxrl/images/07.png' | relative_url }}" alt="从 GRPO 标准差归一化切换到 MaxRL 平均奖励归一化" loading="lazy"></figure>
<div markdown="1">
<p class="visual-note-index">07 / One-line Change</p>
<h2>核心改动：分母从标准差换成均值</h2>

标准 GRPO 常用：

$$A_i^{GRPO}=\frac{r_i-\bar r}{\operatorname{std}(r)+\epsilon}$$

MaxRL 改为：

$$A_i^{MaxRL}=\frac{r_i-\bar r}{\bar r+\epsilon}$$

这正是上面 `1/p` 重加权在组内的可实现形式。若 `K = 0`，即整组都错，则设 advantage 为 0 或跳过该组 loss；此时没有正样本可作为直接学习信号。
</div>
</section>

<section class="visual-note" markdown="1">
<figure><img src="{{ '/assets/posts/video-notes/2026-07-12-maxrl/images/08.png' | relative_url }}" alt="8 个 rollout 中 2 个正确的 MaxRL advantage 示例" loading="lazy"></figure>
<div markdown="1">
<p class="visual-note-index">08 / Worked Example</p>
<h2>8 次采样、2 次正确，为什么是 <code>3</code> 与 <code>-1</code></h2>

令 `r = [1,0,1,0,0,0,0,0]`，则 `r̄ = 2/8 = 0.25`。因此：

$$A_{good}=(1-0.25)/0.25=3,\qquad A_{bad}=(0-0.25)/0.25=-1$$

正确样本获得 3 倍正向权重，错误样本被压低，而且组内均值仍为零：`(2×3 + 6×(-1)) / 8 = 0`。这同时保留了 baseline 降方差的好处。

作为对照，标准 GRPO 使用同一组奖励的 z-score。按实现中常见的总体标准差 `std(r, unbiased=False)`，有 `std(r) = √0.1875 ≈ 0.433`，因此：

$$A_{good}^{GRPO}=0.75/0.433\approx1.73,\qquad A_{bad}^{GRPO}=-0.25/0.433\approx-0.58$$

也就是说，这组样本在标准 GRPO 下约为 `[1.73,-0.58,1.73,-0.58,-0.58,-0.58,-0.58,-0.58]`；在 MaxRL 下则是 `[3,-1,3,-1,-1,-1,-1,-1]`。MaxRL 将正、负 advantage 同比例放大约 `√3 ≈ 1.73` 倍，使这道低成功率题对整体训练产生更强的更新信号。
</div>
</section>

<figure><img src="{{ '/assets/posts/video-notes/2026-07-12-maxrl/images/09.png' | relative_url }}" alt="MaxRL 在训练器中的 advantage 伪代码" loading="lazy"></figure>
<div markdown="1">
<p class="visual-note-index">09 / Implementation</p>
<h2>训练器里应改 advantage，而不是缩放 reward</h2>

```python
r = rewards.float()
mean_r = r.mean()
advantages = (r - mean_r) / (mean_r + 1e-8) if mean_r > 0 else torch.zeros_like(r)
```

如果框架在之后又强制做组内 z-score，必须关闭或改写它。只把正确奖励从 `1` 放大到 `10` 或 `100` 没用：线性 reward rescale 会被 z-score 抵消，得不到 MaxRL 所需的相对权重。
</div>
</section>

<figure><img src="{{ '/assets/posts/video-notes/2026-07-12-maxrl/images/10.png' | relative_url }}" alt="MaxRL 的适用边界与训练监控项" loading="lazy"></figure>
<div markdown="1">
<p class="visual-note-index">10 / Guardrails</p>
<h2>它不是万能 reward 技巧</h2>

MaxRL 的理论前提是可靠的二元正确性奖励；连续 reward-model 分数不能直接当作严格的 MaxRL。全错组没有梯度，因此基础模型需要已有一定成功率，或配合探索、课程学习、拒绝采样等办法。

低成功率题会带来较大的正向权重。训练时建议开启梯度裁剪，并持续监控每组 `K`、`mean_r`、advantage 范数和全错组比例。若使用 PPO ratio / clipping，advantage 替换仍是核心改动，但严格无偏的理论对应 fully on-policy 设置。
</div>
</section>

<div markdown="1">
<p class="visual-note-index">11 / Takeaway</p>
<h2>一行代码背后，是训练目标的切换</h2>

`std(r) → r̄` 并不是一次普通的归一化微调：它把偏好从只盯住 `pass@1`，变成近似最大似然、重视全部 `pass@k`。从策略梯度看，MaxRL 最终仍是用 advantage 推高正确轨迹、压低错误轨迹；不同之处是，它不再让低成功率题目在平均目标里悄悄消失。

对于已有可靠 verifier、并且希望把“偶尔成功”变成“稳定成功”的后训练任务，这是一条值得单独实验和监控的估计器分支。
</div>
</section>

## 实现速查

```python
if estimator == "grpo":
    adv = (r - r.mean()) / (r.std(unbiased=False) + eps)
elif estimator == "maxrl":
    mean_r = r.mean()
    adv = (r - mean_r) / (mean_r + eps) if mean_r > 0 else torch.zeros_like(r)
```

使用前确认：奖励来自二元 verifier；同一 prompt 的 rollout 被正确分组；后续没有覆盖 MaxRL advantage 的 z-score；并且训练日志包含全错组比例和梯度范数。
</section>

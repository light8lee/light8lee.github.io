# top p、top k和temperature

大语言模型在生成每一个词（token）时，都会输出一个概率分布。如果每次都选概率最高的词（贪心解码），生成结果会非常重复。**Top-k 采样**和 **Top-p 采样**正是用来增加多样性的两种主流策略。

---

## 1. Top-k 采样

**核心思想**：每一步只保留概率最高的 K 个 token，在这 K 个 token 上重新归一化概率，然后从中随机采样。

**步骤**：

1. 模型给出词表上所有 token 的概率 PP。
    
2. 选出概率最高的 K 个 token。
    
3. 将这 K 个概率重新归一化，让它们之和为 1。
    
4. 从这 K 个 token 中按新概率随机抽取一个作为输出。

**优缺点**：

- **优**：简单直接，能有效截断长尾的噪声 token。
    
- **劣**：固定的 K 无法适应不同概率分布的形状。当概率分布很集中时，可能截掉一些合理选项；当分布很平坦时，可能包含太多低概率选项。
    

---

## 2. Top-p 采样（核采样，Nucleus Sampling）

**核心思想**：每一步选取一个**最小 token 集合**，使得集合的累积概率刚好超过阈值 pp（如 0.9），然后在这个集合内重新归一化并采样。

**步骤**：

1. 将 token 按概率从高到低排序。
    
2. 从最高概率开始逐个累加，直到累计概率 ≥ p。
    
3. 保留下来的 token 构成“核”集合 V(p)。
    
4. 在这个集合上重新归一化概率，并随机采样。

- **劣**：阈值 p 需要调参，极端情况可能保留过多低概率 token。
    

---

## 3. Top-k 与 Top-p 对比

| |**Top-k 采样**|**Top-p 采样（核采样）**|
|---|---|---|
|**筛选依据**|固定数量 K|动态累积概率阈值 p|
|**候选集大小**|始终为 K|随分布变化|
|**对分布的适应性**|弱，无法感知分布形状|强，能根据分布自动调节|
|**常见取值**|K=40, 50, 100|p=0.9, 0.95|
|**典型问题**|固定 K 易截断合理选项或引入低概率词|p 值过大可能保留过多噪声|

**实际中常常两者结合使用**：先用 Top-p 过滤，再设置一个 Top-k 作为最大数量上限，避免候选集过大。

---

## 4. 代码示例（PyTorch 风格）

```python

import torch
import torch.nn.functional as F
def sample_top_k(logits, k):
    """Top-k 采样"""
    # 取 top-k 的值与索引
    topk_values, topk_indices = torch.topk(logits, k, dim=-1)
    # 将其他位置置为 -inf
    min_topk = topk_values[:, -1].unsqueeze(-1)
    logits = torch.where(logits < min_topk, torch.tensor(float('-inf')), logits)
    # 重新归一化
    probs = F.softmax(logits, dim=-1)
    return torch.multinomial(probs, 1).squeeze(-1)
def sample_top_p(logits, p):
    """Top-p（核）采样"""
    # 按概率降序排序
    probs = F.softmax(logits, dim=-1)
    sorted_probs, sorted_indices = torch.sort(probs, descending=True, dim=-1)
    # 累积概率
    cumsum = torch.cumsum(sorted_probs, dim=-1)
    # 找到累积概率超过 p 的最后一个索引
    mask = cumsum - sorted_probs > p
    sorted_probs[mask] = 0
    # 重新归一化
    sorted_probs /= sorted_probs.sum(dim=-1, keepdim=True)
    # 采样（从原始索引）
    sampled_sorted_idx = torch.multinomial(sorted_probs, 1)
    return sorted_indices.gather(-1, sampled_sorted_idx).squeeze(-1)
```


Temperature 与 Top-k、Top-p 并不是互相替代的关系，而是**作用于不同阶段的、可以组合使用的“调节旋钮”**。它们在文本生成流程中按先后顺序共同决定了“随机性”与“质量”。

## 1. Temperature：重塑“原始概率”的形状

**Temperature (τ) 是在 Softmax 之前应用的缩放因子**，它不直接决定选哪些词，而是改变概率分布的平滑度。

- **τ=1**：原始概率。
    
- **τ>1**（如 1.2）：分布更平坦，低概率词获得更多机会，**增加多样性**。
    
- **0<τ<1**（如 0.7）：分布更尖锐，高概率词更突出，**趋向确定性**。
    

---

## 2. Top-k / Top-p：筛选“候选词”的范围

**Top-k 和 Top-p 是在概率分布已经确定之后，对候选 token 进行硬截断**。

- **Top-k**：固定保留概率最高的 k 个 token。
    
- **Top-p**：动态保留累积概率刚好超过 p 的最小 token 集合。
    

两者都是通过将其他 token 概率置 0 并重新归一化，来剔除长尾噪声。

---

## 3. 三者如何协作？（生成流程中的顺序）

实际生成一个 token 的标准流程是：

1. **模型前向** → 得到 logits (原始分数)
    
2. **Temperature 缩放** → logits/τ → 经 Softmax 得到平滑或尖锐的概率分布
    
3. **Top-k / Top-p 截断** → 在得到的分布上，只保留部分 token，重新归一化
    
4. **采样** → 从最终分布中随机抽取一个 token
    

**关键互动**：

- Temperature 改变了分布的形状，**进而影响 Top-p 的“核”大小**。
    
    - τ 大 → 分布平坦 → 要累积到 p (如 0.9) 需要更多 token → Top-p 的候选集变大。
        
    - τ 小 → 分布尖锐 → 可能前两个 token 就超过 p → 候选集变得极小，接近贪心。
        
- Temperature 也会影响 Top-k 的效果：τ 大时，Top-k 可能仍然包含很多低概率词；τ 小时，Top-k 几乎等价于直接选最高那个。
    

---

## 4. 形象类比

- **Temperature 是“调色温”**：把整张照片变暖或变冷，整体改变氛围。
    
- **Top-p 是“动态裁切”**：根据画面内容，自动裁掉无关的杂乱背景。
    
- **Top-k 是“固定裁切”**：不管内容如何，只保留画面中央固定大小的区块。
    

实际图像处理流程会是：先调色温（Temperature），再裁切（Top-p/Top-k），最后取图（采样）。

---

## 5. 常见组合与推荐配置

几乎所有语言模型 API 和生成框架（HuggingFace、vLLM 等）都支持同时配置这三个参数。

**典型组合**：

|场景|Temperature|Top-p|Top-k|说明|
|---|---|---|---|---|
|事实问答/翻译|0.2~0.5|0.9|50|低温度减少随机，top-p 防致命错误|
|创意写作|0.8~1.0|0.95|50~100|高温度激发多样性，高 top-p 拓宽候选|
|代码生成|0.2~0.6|0.9|50|相对保守但保留备选|
|对齐训练（PPO/GRPO rollout）|0.7~1.0|0.9|50|需要较强探索性，常设 τ=1.0 以匹配 π_θold 的真实分布|

---

## 6. 与强化学习对齐的关系

回到我们一直在讨论的 RLHF、PPO、GRPO：

- 在**rollout 生成回答**时，需要从 `π_θold` 采样。**Temperature、Top-k、Top-p 正是控制这一采样的超参数**。
    
- 如果 Temperature 过低或 Top-p 过窄，采样结果将极度趋同（缺乏探索），导致 GRPO 的组内优势失效（所有回答的奖励几乎一样，优势全为零）。
    
- 如果 Temperature 过高或 Top-p 过宽，可能产生大量语法混乱但偶尔拿高分的“作弊文本”，污染训练。
    

因此，这些采样参数是连接“概率模型”与“有效探索”的重要工程设计，**必须与 KL 惩罚系数 β、组大小 G 配合调试**。

---

## 总结

| |**Temperature**|**Top-k**|**Top-p (核采样)**|
|---|---|---|---|
|**作用阶段**|Softmax 前 (logits 缩放)|Softmax 后 (概率截断)|Softmax 后 (动态截断)|
|**作用效果**|改变分布的整体平滑度|固定数量候选|动态累积概率候选|
|**影响多样性方式**|连续调节|硬性数量限制|基于分布形状的软限制|
|**核心参数**|τ (典型 0.5~1.2)|k (典型 40~100)|p (典型 0.9~0.95)|
|**依赖性**|独立|可与 Temperature、Top-p 叠加|可与 Temperature、Top-k 叠加|

三者**串行协作**，Temperature 打底，Top-p/Top-k 截尾，共同实现可控的随机性。

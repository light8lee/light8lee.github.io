2026 年 6 月 16 日，普林斯顿大学的 Haozhe Chen、Karthik Narasimhan、Zhuang Liu 发布了预印本《CEO-Bench: Can Agents Play the Long Game?》。这篇工作问的不是 agent 会不会写代码、会不会做一次性的任务，而是更难的一件事: 如果把一个公司交给 agent 连续经营 。

<h2>Agent 会干活，不等于会带公司</h2>

很多 agent 现在已经会写代码、查资料、调工具了，但 CEO-Bench 提醒我们，会干活不等于会带公司。真正难的不是单次执行，而是在长周期里持续把方向带对。
</div>
</section>

<div markdown="1">
<p class="visual-note-index">002 / Video Notes</p>
<h2>CEO-Bench / 2026-06-16</h2>

二零二六年六月十六日，普林斯顿大学发布预印本 CEO-Bench。它不再问 agent 会不会做一个任务，而是问它能不能在长期、不确定、持续变化的环境里稳住系统目标。
</div>
</section>

<section class="visual-note" markdown="1">
<figure>
<img src="{{ '/assets/posts/video-notes/2026-06-19-ceobench/images/03.png' | relative_url }}" alt="它测的是 steering intelligence" loading="lazy">
</figure>
<div markdown="1">
<p class="visual-note-index">003 / Video Notes</p>
<h2>它测的是 steering intelligence</h2>

这篇工作把问题从 task intelligence 推到 steering intelligence。你不是只要把某一步做对，而是要把很多周的局部决策，最后收束成一个一致而且可持续的方向。
</div>
</section>

<section class="visual-note" markdown="1">
<figure>
<img src="{{ '/assets/posts/video-notes/2026-06-19-ceobench/images/04.png' | relative_url }}" alt="起点: 100 万美元，目标: 活过 500 天" loading="lazy">
</figure>
<div markdown="1">
<p class="visual-note-index">004 / Video Notes</p>
<h2>起点: 100 万美元，目标: 活过 500 天</h2>

在 CEO-Bench 里，agent 拿着一百万美元启动资金去经营一个 startup，连续跑五百个模拟日。它面对的是公司经营，不是一次性的 puzzle，所以现金、节奏和方向都要同时兼顾。
</div>
</section>

<section class="visual-note" markdown="1">
<figure>
<img src="{{ '/assets/posts/video-notes/2026-06-19-ceobench/images/05.png' | relative_url }}" alt="不是点按钮，而是用 Python 经营" loading="lazy">
</figure>
<div markdown="1">
<p class="visual-note-index">005 / Video Notes</p>
<h2>不是点按钮，而是用 Python 经营</h2>

公开仓库把动作面暴露成 Python 包 novamind_api。也就是说，agent 不是被锁死在几个按钮里，而是可以自己写脚本、查数据库、组织流程，这更接近真实部署里的 coding agent 形态。
</div>
</section>

<figure>
<img src="{{ '/assets/posts/video-notes/2026-06-19-ceobench/images/06.png' | relative_url }}" alt="19 张业务表，34 个动作" loading="lazy">

<div markdown="1">
<p class="visual-note-index">006 / Video Notes</p>
<h2>19 张业务表，34 个动作</h2>

这个 benchmark 的输入面很像真实公司的 analytics stack。十九张业务表，加上三十四个可组合动作，覆盖定价、增长、产品、运营、信息获取和公开沟通，逼着 agent 先分析，再决策。
</div>
</section>

<section class="visual-note" markdown="1">
<figure>
<img src="{{ '/assets/posts/video-notes/2026-06-19-ceobench/images/07.png' | relative_url }}" alt="真正难的是延迟反馈" loading="lazy">
</figure>
<div markdown="1">
<p class="visual-note-index">007 / Video Notes</p>
<h2>真正难的是延迟反馈</h2>

为什么它难？因为这里不是立刻反馈的游戏。广告可能今天花钱，几周后才带来客户；研发有随机完成时间；企业销售要跨多轮谈判；竞争对手和用户偏好还会一路漂移。
</div>
</section>

<figure>
<img src="{{ '/assets/posts/video-notes/2026-06-19-ceobench/images/08.png' | relative_url }}" alt="结果: 多数模型连本金都守不稳" loading="lazy">
</figure>
<div markdown="1">
<p class="visual-note-index">008 / Video Notes</p>
<h2>结果: 多数模型连本金都守不稳</h2>

结果很直接。论文摘要写得很清楚，只有 Claude Opus 四点八和 GPT 五点五，能在最好的一次 run 里高于一百万美元起始资金，而且它们都还不能稳定盈利。
</div>
</section>

<section class="visual-note" markdown="1">
<figure>
<img src="{{ '/assets/posts/video-notes/2026-06-19-ceobench/images/09.png' | relative_url }}" alt="best run 高，不代表稳定性够" loading="lazy">
</figure>
<div markdown="1">
<p class="visual-note-index">009 / Video Notes</p>
<h2>best run 高，不代表稳定性够</h2>

官网又补了一层更重要的信息。Claude Fable 五、Claude Opus 四点八和 GPT 五点五的最佳 run 都能高于起始资金，但只有 Claude Fable 五不止一次跑在线上。也就是说，重复稳定性依然很差。
</div>
</section>

<section class="visual-note" markdown="1">
<figure>
<img src="{{ '/assets/posts/video-notes/2026-06-19-ceobench/images/10.png' | relative_url }}" alt="会分析，不等于会持续 steering" loading="lazy">

<div markdown="1">
<p class="visual-note-index">010 / Video Notes</p>
<h2>会分析，不等于会持续 steering</h2>

作者提到，最强的 agent 已经会自己写复杂代码，比如模拟 customer cohort 预测现金流，或者挖谈判历史找隐藏偏好。问题不只是会不会分析，而是能不能把分析、下注、调整和复盘串成长期闭环。
</div>
</section>

<section class="visual-note" markdown="1">
<figure>
<img src="{{ '/assets/posts/video-notes/2026-06-19-ceobench/images/11.png' | relative_url }}" alt="下一代 agent 要补 steering 层" loading="lazy">
</figure>
<div markdown="1">
<p class="visual-note-index">011 / Video Notes</p>
<h2>下一代 agent 要补 steering 层</h2>

所以 CEO-Bench 真正揭示的，不是 agent 不会做事，而是它们离长期 steering 一个复杂系统还差一层关键能力。下一代 agent 要补的，是把目标、状态、分析、行动和复盘稳定连起来的掌舵层。
</div>
</section>

# SCPO 小红书修订说明

## 07 创新点 4：信用分配顺序重排

原版问题：只说“要重排”，没有讲清楚怎么重排、为什么重排。

新版逻辑：

1. 为什么要重排
   - SCPO 的信用分配不是无限的。一个参考步骤最多只能被匹配一次。
   - 如果按失败轨迹的时间顺序贪心匹配，早期通用动作会先占坑。
   - 例如“打开网页”“搜索框”“输入关键词”在很多轨迹里都相似，但它们信息量低。
   - 后面真正决定任务成功的动作，如“选择正确航班”“核对价格”“进入支付页”，可能反而没有机会拿到信用。

2. 怎么重排
   - 先构建失败步骤和成功参考步骤的语义相似度矩阵。
   - 对候选匹配做“长度/位置平衡”的顺序安排，而不是简单按时间从前到后处理。
   - 直观理解：不要让开头模板动作先把参考轨迹的位置消耗完，要让前、中、后段都有机会申请信用。
   - 最后仍然要经过单调约束：匹配必须沿参考轨迹向前，且每个参考位置最多用一次。

3. 小红书图里要画什么
   - 左侧：时间顺序贪心。绿色 token 都被早期 Start/Search 抢走，关键动作灰掉。
   - 中间：相似度矩阵 + “重排候选队列”。
   - 右侧：重排后。信用分布到 Search、Select flight、Checkout 等关键进展。
   - 结论短句：重排不是改奖励，而是改“谁先申请信用”。

## 08 创新点 5：只平移 step advantage

原版问题：例子不够细，读者不容易看出 SCPO 到底改了哪里。

新版例子：买机票失败轨迹

成功参考：
1. 打开携程
2. 搜索北京到上海
3. 选择 08:00 航班
4. 进入支付页
5. 支付成功

失败轨迹：
1. 打开携程
2. 搜索北京到上海
3. 选择 08:00 航班
4. 进入支付页
5. 支付失败

计算直觉：

- 原始 episode reward：失败，所以 AE 仍然是负的，例如 `AE = -0.5`。
- SCPO 发现前 4 步和成功参考语义一致，所以这些步骤得到 step credit。
- 例如“选择 08:00 航班”的 `AS` 从 `-0.5` 被拉到 `-0.2`。
- 最终 advantage：`A = AE + ω AS`。
- SCPO 只让局部正确步骤少挨罚，不改变“这条轨迹失败了”的事实。

图里要画：

- 上半部分：成功/失败两条买机票轨迹，前 4 步重合为绿色，第 5 步分叉。
- 中部：三行数字卡。
  - 原始：`AE = -0.5`
  - SCPO：`AS: -0.5 -> -0.2`
  - 最终：`A: -0.75 -> -0.60`
- 下半部分：一句话结论。
  - “失败仍惩罚，但别过度惩罚正确步骤”
